# Custom Bundle

Use this mode when the Gemini Gem goal does not fit one of the preset bundles cleanly.

## First step for Custom mode

Ask the user to define the Gem's job clearly.

Useful follow-up questions:

1. What kinds of tasks should the Gem be good at?
2. Should the Gem stay reusable across repos, or is it specific to this repo?
3. Which languages, frameworks, or workflows matter most?
4. Should repo-local skills be excluded unless absolutely necessary?

## Core rules for Custom mode

- start with the smallest useful skill set
- include hard dependencies when a selected skill needs them
- avoid repo-local skills unless the Gem is intentionally repo-specific
- give the user a concrete final list of skills plus a paste-ready instruction block
