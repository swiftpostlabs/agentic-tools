# Maintenance Checklist

- The change surface was inspected before editing skills.
- Branch diff or local modified files were used when available.
- Each changed repo behavior is mapped to one owning skill or intentionally left uncovered with a reason.
- Top-level routing was refreshed if the skill catalog changed.
- Stale commands, examples, package managers, or library references were removed.
- Shareability metadata still matches the skill's real portability and dependencies.
- Touched skill files were checked for diagnostics and catalog discovery.
